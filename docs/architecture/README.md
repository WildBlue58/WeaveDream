## 架构概述

项目采用模块化架构，主要分为以下几个层次：

1. **UI 组件层** (`components/`) - 可视化的 UI 组件
2. **物料组件层** (`materials/`) - 可拖拽的物料组件
3. **状态管理层** (`stores/`) - Zustand 状态管理
4. **工具函数层** (`hooks/`, `utils/`) - 可复用的逻辑

## 核心概念

### 组件树

- 使用树形结构组织组件
- 通过 `parentId` 和 `children` 建立父子关系
- Page 组件作为根节点

### 属性编辑

- 每个组件有独立的属性编辑器
- 通过注册表模式管理编辑器映射
- 属性变更实时更新到组件树

### 状态管理

- 使用 Zustand 管理全局状态
- 支持撤销/重做功能
- 支持复制/粘贴功能
